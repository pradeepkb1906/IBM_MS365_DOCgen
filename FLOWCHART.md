## 4. KB image enrichment — 3-tier fallback (the "wow" feature)

```mermaid
flowchart TD
    START["_kb_enrich_sections called with<br/>sections + file_ids + auth"] --> FETCH["_fetch_attachment_bytes<br/>GET /api/v1/files/{id}/content<br/>Returns [(filename, ext, bytes), ...]"]
    FETCH --> SPLIT{split by ext}
    SPLIT -->|.pdf| PDFS[PDFs]
    SPLIT -->|.docx/.pptx/.xlsx| OFF[Office files]

    subgraph Tier1a["Tier 1a — Office zipfile extraction"]
        OFF --> OX["_extract_office_images<br/>• per-slide context via slide rels map<br/>• scans word/media, ppt/media, xl/media,<br/>  word/embeddings, ppt/embeddings"]
        OX --> POOL[(KB image pool)]
    end

    subgraph Tier1b["Tier 1b — PDF page snapshots (pypdfium2 / fitz)"]
        PDFS --> PX["_pdf_pages_as_images<br/>• pypdfium2: page.render(scale=1.5).to_pil() → PNG<br/>• fallback to fitz if pypdfium2 missing<br/>• textpage extract for BM25 seed<br/>• up to 20 pages per PDF"]
        PX --> POOL
    end

    POOL --> PARMAP["For each section in parallel<br/>(asyncio.gather)"]
    PARMAP --> BM25["_bm25_prefilter<br/>token overlap: section title+bullets vs caption_seed<br/>→ top-5 candidates"]
    BM25 --> PLC["_plan_c_match<br/>Builds Claude message with 5 image blocks<br/>+ section title + bullets<br/>→ Claude returns {best_idx, score, caption}"]

    PLC --> GATE{"score ≥ threshold?<br/>(default 50)"}
    GATE -->|yes| COMP["_compress_kb_image<br/>• PIL resize to 1000px longest edge<br/>• progressive JPEG q=82<br/>• typical 5-75× smaller"]
    COMP --> STAMP["section._kb_match = {<br/>  image_bytes, image_ext='jpg',<br/>  source_file, score, caption<br/>}"]

    GATE -->|no| WIKI{"kb_wikipedia_fallback = True?"}
    WIKI -->|yes| WK["_wikipedia_image_for<br/>(a) GET /api/rest_v1/page/summary/{title}<br/>(b) fallback: GET /search/title?q=...<br/>• try 3 query variants (full/stripped/2-words)<br/>• skip .svg sources (no rasterizer)"]
    WK --> WOK{"got image?"}
    WOK -->|yes| COMP
    WOK -->|no| NONE["no _kb_match → 100% text"]
    WIKI -->|no| NONE

    STAMP --> EMIT["Emit status: '👁️ 50/50 layout: K/N — A from attachment, W from Wikipedia'"]

    classDef tier1 fill:#E8F5E9,stroke:#24A148
    classDef tier2 fill:#E0F7FA,stroke:#007D79
    classDef tier3 fill:#FFF3E0,stroke:#FF832B
    class Tier1a,Tier1b tier1
    class PLC tier2
    class WK tier3
```

**Keys to read this:**

- **Tier 1a** always runs (if Office attachment present). Cheap, zero-deps.
- **Tier 1b** only fires if `pypdfium2` or `fitz` is importable on the runtime Python. Both on localhost. Usually absent on Beta — that tier silently no-ops.
- **Claude Vision rank** (Plan C) is the gate. Scores 0-100, threshold default **50** (user-tuned from initial 95).
- **Tier 3 Wikipedia** only fires when Plan C produced nothing above threshold.
- Final `section._kb_match` drives the 50/50 layout in both builders AND the HTML iframe.

## 8. Valve summary (tunable knobs)

| Valve | Default | What it controls |
|---|---|---|
| `enable_kb_vision_layout` | True | Turn the whole 50/50 KB feature on/off |
| `kb_vision_score_threshold` | **50** | Min Claude Vision score to accept a KB image. Raise for stricter, lower for more images |
| `kb_max_candidates_per_section` | 5 | How many BM25-top candidates to send to Claude Vision per section |
| `kb_plan_b_enabled` | True | Try Anthropic native PDF document block first (falls through if Bedrock rejects) |
| `kb_image_max_dim` | 1000 px | Longest-edge cap after resize |
| `kb_image_jpeg_quality` | 82 | JPEG Q after resize (clamped 40-95) |
| `kb_preview_lazy_hydration` | True | `<img loading="lazy" decoding="async">` browser hints |
| `kb_wikipedia_fallback` | True | Tier 3 Wikipedia REST fetch when KB empty |
| `kb_wikipedia_timeout` | 5 s | Per-call Wikipedia timeout |
| `vision_rank_model` | `claude-opus-4-6` | Bedrock Claude via litellm |
| `disable_image_enrichment` | True | Legacy web-image enrichment killswitch (panic button) |

## 10. Cradle-to-grave example: "make a 10-slide PPTX about IBM cloud + include my PDF"

```mermaid
sequenceDiagram
    autonumber
    participant U as User
    participant O as OWUI chat
    participant L as LLM (Claude Opus 4.6)
    participant T as Tool (Tools class)
    participant B as Bedrock API
    participant W as Wikipedia
    participant P as pypdfium2

    U->>O: "10 slides about IBM cloud — use attached PDF"
    O->>L: system prompt + user msg + __files__=[{id: ...pdf}]
    L->>T: prepare_content_smart(query="IBM cloud strategy")
    T->>O: GET /api/v1/files/{id}/content (Bearer JWT)
    O-->>T: pdf bytes
    T-->>L: text_chunks + images (from pdf pages)
    L->>T: assemble_document(format=pptx, sections_json=[...10 sections], __files__=[pdf])
    T->>P: pypdfium2.PdfDocument → 20 pages → PNG
    T->>T: _extract_office_images (no office attachments here)
    T->>T: _bm25_prefilter per section → top-5 candidates
    loop for each section (parallel)
        T->>B: POST /chat/completions with 5 image blocks + section title
        B-->>T: {"best_idx": 2, "score": 78, "caption": "..."}
        alt score ≥ 50
            T->>T: _compress_kb_image (PIL resize + JPEG q82)
        else no good match
            T->>W: GET /page/summary/{section title}
            W-->>T: thumbnail URL
            T->>W: GET thumbnail.jpg
            W-->>T: image bytes → compress
        end
    end
    T->>T: _build_and_render_pptx<br/>→ 10 slides × (shapes + chart_graphic_frame + kb image + source line)
    T-->>O: HTMLResponse(content=html, headers={Content-Disposition: inline})
    O-->>U: iframe preview with Prev/Next/Zoom/Download + 50/50 layout per section
```
